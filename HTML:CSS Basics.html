Hyperlinks

The power of the web is that pages can lead to other pages. When we click a link on a web page, it takes us to another page. This link is called a hyperlink

Hyperlinks are created using anchor elements, which generally look like:

Google This is the hyperlink leading to google's homepage

Inside the opening a tag there is href, which stands for "reference". This is called an attribute . Attributes like href describe the properties of HTML elements. In this case, the href attribute is the target URL that the link will open. The content inside the anchor element is the text that users see displayed on the page.

Syntax for making hyperlinks :

  • There is a space between a and href
  • There are no spaces aroung the =
  • The website has two " around it
  • There are no spaces between the href attribute and the opening tag

Images

An image element does not need a closing tag (also called void elements)

The source attribute, src, is like the href of a link - it is the URL of the image we want to display. For now, our images will need to be hosted online, which means that the URL will need to start with http:// or https://. The alt attribute stands for "alternative description", which is important for people who use screen readers to browse the web. This is the text that will show up lieu of the actual image.

A Guide to Paths

In order to display a local image, we need to be able to write a path

A path is a way of describing where a file is stored

As a web developer, we need to consider two essential domains for paths:

  1. Paths to find files on your computer, local files
  2. Paths to find files on other computers, external files

Local Paths

Computers have folders (also called "directories"). Operating systems like Windows, Mac and Linux organize all of the files into a tree of directories called a file system . There's a top-most directory, often called the root , that contains all of the other directories. Within the root, there are files and directories. Withing those directories are more files and directories and so on.

Every file has an address, which we call the "path". An absolute path is written in relation to the computer's root directory. For instance, a file in the Documents folder on a Mac has a path that looks like: "/Users/Hetansh/Documents/file.txt"

file.txt is stored inside Documents/. Users/, Hetansh/ and Documents/ are all names of directories. Documents/ lives inside Hetansh/ and Hetansh/ lives inside Users/. Users/ is inside the root directory, which is represented by the first "/". The rest of the "/" are used to separate directories.

When we open an HTML file in the browser, we're seeing the absolute path to the file on the computer.

file:///Users/hetanshmehta/Desktop/HTML:CSS%20Basics.html

file:// is called the protocol and the rest is the absolute path to index.html. This URL will only work for you on your computer. As no one else has your file system, this URL is unique to your computer.

External Paths

The process of leading a website from a URL like https://www.google.com mimics opening an HTML file that is saved on the computer. Every Website starts with an HTML file. It just so happens that when we visit a website, the HTML file that we want to open lives on a different computer. The computer responsible for giving the website's files is called a server

Pointing your browser to hrrps://www.google.com sends a request to Google's server for the HTML file (and others) that your computer needs to load the Google website. We can think of google.com as the root path of google/s server (computer) that anyone can access. The google server exposes the files to the web, which means that they make them available to anyone who wants them. Servers have external path that anyone can access and is the reason why the web works.

There are different protocols for serving files, the most common of which on the web are HTTP and HTTPS. Opening the file on your own computer uses the FILE protocol.
